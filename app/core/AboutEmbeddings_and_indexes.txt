 import pinecone
 import os

 pinecone.init(
     api_key=os.environ.get("PINECONE_API_KEY"),
     environment=os.environ.get("PINECONE_ENVIRONMENT")
 )

 index_name = "rag-pdf-index" #(da inserire in .env)

 # crea index solo se non esiste
 if index_name not in pinecone.list_indexes():
     pinecone.create_index(
         name=index_name,
         dimension=1536,  # 1536 corrisponde a text-embedding-3-small usato di default in OpenAIEmbeddings (embeddings_model = OpenAIEmbeddings())
         metric="cosine"
     )



 1️⃣ Cosa sono i modelli text-embedding-3-small e text-embedding-3-large

 Gli embedding trasformano un testo in un vettore numerico che cattura il significato semantico del testo.
 Pinecone poi usa questi vettori per fare similarity search (cioè trovare chunk di testo simili a una query).

 Modello	Dimensione	Velocità	Costo	Precisione / Qualità
 text-embedding-3-small	1536	veloce	basso	buona per documenti piccoli/medi
 text-embedding-3-large	3072	più lento	più alto	migliore, più preciso per documenti complessi o lunghi

 Small: ottimo per prototipi o PDF di dimensioni medie, e per un numero limitato di utenti.

 Large: utile se hai PDF molto lunghi o complessi, e vuoi risultati di retrieval più accurati.

"text-embedding-3-small"  e uguale e' deve  deve corrispondere all' index dimension=1536 quando andiamo a creiamo l'index.
